## 2. File Processing Bug Fixed

### Problem Identified:
Files were stuck at "processing" status for hours.

### Root Cause:
- **Upload hook** (`use-file-upload.ts`) uploaded files to bucket: `"business-files"`
- **Processors** (`utils.ts`) tried to download from bucket: `"uploads"`
- Result: Files couldn't be found for processing

### Fix Applied:
```typescript
// src/lib/processors/utils.ts
// Changed from:
.from("uploads")
// Changed to:
.from("business-files")
```

### Commit:
```
fix: Correct storage bucket name from 'uploads' to 'business-files'
```

## 4. File Processing Architecture Summary

### Upload Flow:
```
User Upload → Supabase Storage (business-files bucket) → DB record (uploads table) → POST /api/process
```

### Processing by File Type:
| Type | Processor | AI Model | Time |
|------|-----------|----------|------|
| Image | `image.ts` | Gemini 1.5 Flash Vision | 5-15s |
| Excel/CSV | `spreadsheet.ts` | Embeddings only | 10-30s |
| PDF | `pdf.ts` | pdf-parse + Embeddings | 10-30s |
| Audio | `audio.ts` | Gemini 1.5 Flash Audio | 15-60s |

### Processing Status Flow:
```
pending → processing → completed ✓
                    ↓
                    failed ✗
```

## 9. How to Reprocess Stuck Files

### Option 1: UI (After deployment)
1. Go to `/app/archivos`
2. Click ⋮ menu on stuck file
3. Click "Reprocesar"

### Option 2: SQL Reset
```sql
UPDATE uploads
SET processing_status = 'pending', processed = false
WHERE processing_status = 'processing'
  AND updated_at < NOW() - INTERVAL '30 minutes';
```

### Option 3: Browser Console
```javascript
fetch('/api/process', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ uploadId: 'YOUR_UPLOAD_ID' })
}).then(r => r.json()).then(console.log)
```
